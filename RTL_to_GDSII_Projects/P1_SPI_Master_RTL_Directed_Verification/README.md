## Design Description

### SPI Protocol — Mode 0

SPI (Serial Peripheral Interface) is a synchronous, full-duplex, master-slave protocol. The master generates all clocking and controls chip-select. Four signals are used:

| Signal | Direction | Function |
|--------|-----------|----------|
| MOSI | Master → Slave | Serial data out, MSB first |
| MISO | Slave → Master | Serial data in, tri-states when CS=HIGH |
| SCK | Master → Slave | Serial clock, generated by master only |
| CS_N | Master → Slave | Active-LOW chip select |

Mode 0 (CPOL=0, CPHA=0): SCK idles LOW; data is sampled on the rising edge and shifted on the falling edge.

### FSM Architecture

The design implements a 5-state Moore FSM. All outputs are registered (no combinational glitches on CS_N or SCK).

```
IDLE ──(start=1)──► ACTIVATE_CS ──► SHIFT ──(bit_cnt==7 & sck_en)──► DEACTIVATE_CS ──► DONE ──► IDLE
                                      ▲                                                              
                                      └──────────── (bit_cnt < 7 & sck_en) ──────────────────────────
```

| State | CS_N | SCK | Key Action |
|-------|------|-----|------------|
| IDLE | 1 | CPOL | Wait for `start`; latch `mosi_data` |
| ACTIVATE_CS | 0 | CPOL | Assert CS; pre-load MOSI with bit 7 |
| SHIFT | 0 | Toggle | Shift out MOSI, shift in MISO on each `sck_en` |
| DEACTIVATE_CS | 1 | CPOL | Deassert CS; latch `shift_reg_in` → `miso_data` |
| DONE | 1 | CPOL | Pulse `done` for 1 cycle; return SCK to idle |

### Clock Divider

An internal counter generates `sck_en` every `CLK_DIV/2` system clock cycles. `sck_en` triggers SCK toggling inside the SHIFT state.

```
sys_clk  = 50 MHz  (period = 20 ns)
CLK_DIV  = 4
SCK      = sys_clk / CLK_DIV = 12.5 MHz
```

## RTL — spi_master.sv

```systemverilog
// -------------------------------------------------
// spi_master.sv — Parameterised SPI Master FSM
// -------------------------------------------------
module spi_master #(
  parameter DATA_WIDTH = 8, // bits per transfer: 4, 8, or 16
  parameter CLK_DIV    = 4,  // sys_clk / CLK_DIV = SCK frequency
  parameter CPOL       = 0, // 0=idle LOW
  parameter CPHA       = 0 // 0=sample 1st edge
)(
  input  logic                  clk,
  input  logic                  rst_n, // async active-low reset
  input  logic                  start, // pulse high to begin
  input  logic [DATA_WIDTH-1:0] mosi_data, // data to send
  output logic                  done,
  output logic                  cs_n,
  output logic                  sck,
  output logic                  mosi,
  input  logic                  miso,
  output logic [DATA_WIDTH-1:0] miso_data
);

  // State definition
  typedef enum logic [2:0] {
    IDLE          = 3'd0,
    ACTIVATE_CS   = 3'd1,
    SHIFT         = 3'd2,
    DEACTIVATE_CS = 3'd3,
    DONE          = 3'd4
  } state_t;

  state_t state, next_state;

  // Internal signals
  logic [$clog2(CLK_DIV)-1:0]    clk_cnt;       // clock divider counter
  logic                           sck_en;         // SCK toggle enable
  logic [$clog2(DATA_WIDTH)-1:0]  bit_cnt;        // tracks bits shifted
  logic [DATA_WIDTH-1:0]          shift_reg_out;  // data shifting out
  logic [DATA_WIDTH-1:0]          shift_reg_in;   // data shifting in

  // Clock divider — fires sck_en every CLK_DIV/2 sys_clk cycles
  always_ff @(posedge clk or negedge rst_n) begin
    if (!rst_n) begin
      clk_cnt <= '0;
      sck_en  <= 1'b0;
    end else begin
      sck_en <= 1'b0;
      if (clk_cnt == CLK_DIV/2 - 1) begin
        clk_cnt <= '0;
        sck_en  <= 1'b1;
      end else
        clk_cnt <= clk_cnt + 1;
    end
  end

  // FSM next-state logic (combinational)
  always_comb begin
    next_state = state;
    case (state)
      IDLE:          if (start) next_state = ACTIVATE_CS;
      ACTIVATE_CS:   next_state = SHIFT;
      SHIFT:         if (sck_en && bit_cnt == DATA_WIDTH-1)
                       next_state = DEACTIVATE_CS;
      DEACTIVATE_CS: next_state = DONE;
      DONE:          next_state = IDLE;
      default:       next_state = IDLE;
    endcase
  end

  // FSM state register (sequential) — kept separate from output logic
  always_ff @(posedge clk or negedge rst_n) begin
    if (!rst_n) state <= IDLE;
    else        state <= next_state;
  end

  // Output logic — all registered; no glitches on CS_N or SCK
  always_ff @(posedge clk or negedge rst_n) begin
    if (!rst_n) begin
      cs_n         <= 1'b1;
      sck          <= CPOL;
      mosi         <= 1'b0;
      done         <= 1'b0;
      bit_cnt      <= '0;
      shift_reg_out<= '0;
      shift_reg_in <= '0;
      miso_data    <= '0;
    end else begin
      done <= 1'b0; // default: done LOW (1-cycle pulse only)
      case (state)
        IDLE: begin
          cs_n <= 1'b1;
          sck  <= CPOL;
          if (start) begin
            shift_reg_out <= mosi_data;  // latch data as soon as start pulses
            bit_cnt       <= '0;
          end
        end
        ACTIVATE_CS: begin
          cs_n          <= 1'b0;
          mosi          <= shift_reg_out[DATA_WIDTH-1];           // pre-load bit7 to eliminate 1-cycle MOSI lag
          shift_reg_out <= {shift_reg_out[DATA_WIDTH-2:0], 1'b0}; // advance so bit6 is ready for first SHIFT
        end
        SHIFT: begin
          if (sck_en) begin
            sck           <= ~sck;                                        // toggle SCK
            mosi          <= shift_reg_out[DATA_WIDTH-1];                 // drive next bit
            shift_reg_out <= {shift_reg_out[DATA_WIDTH-2:0], 1'b0};      // left-shift out
            shift_reg_in  <= {shift_reg_in[DATA_WIDTH-2:0], miso};       // right-shift in MISO
            if (bit_cnt < DATA_WIDTH-1) bit_cnt <= bit_cnt + 1;
          end
        end
        DEACTIVATE_CS: begin
          cs_n      <= 1'b1;
          miso_data <= shift_reg_in; // latch here — shift_reg_in is complete only after last SCK edge
        end
        DONE: begin
          done <= 1'b1;  // 1-cycle pulse; driver waits on posedge done
          sck  <= CPOL;  // return SCK to idle state
        end
      endcase
    end
  end

endmodule
```

## Testbench — spi_tb_directed.sv

```systemverilog
module spi_tb_directed;
  localparam DATA_WIDTH = 8;
  localparam CLK_DIV    = 4;
  logic clk, rst_n, start, done;
  logic [DATA_WIDTH-1:0] mosi_data, miso_data;
  logic cs_n, sck, mosi, miso;

  spi_master #(
    .DATA_WIDTH(DATA_WIDTH),
    .CLK_DIV   (CLK_DIV),
    .CPOL      (0),
    .CPHA      (0)
  ) dut (.*);

  assign miso = mosi; // loopback: slave echoes master's data back

  initial clk = 0;
  always #10 clk = ~clk; // 50 MHz system clock

  // Reusable task: send one byte and verify MISO matches MOSI (loopback)
  task automatic send_byte(input logic [7:0] data);
    @(posedge clk); mosi_data = data; start = 1;
    @(posedge clk); start = 0;
    @(posedge done); @(posedge clk);
    if (miso_data !== data)
      $error  ("[FAIL] Sent=0x%0h Rcvd=0x%0h", data, miso_data);
    else
      $display("[PASS] Sent=0x%0h Rcvd=0x%0h", data, miso_data);
  endtask

  initial begin
    rst_n=0; start=0; mosi_data=0;
    repeat(4) @(posedge clk); rst_n=1; // hold reset for 4 cycles
    repeat(2) @(posedge clk);
    // Six directed vectors covering: alternating bits, all-1s, all-0s,
    // opposite alternating, LSB-only, MSB-only
    send_byte(8'hA5); // 1010_0101
    send_byte(8'hFF); // 1111_1111
    send_byte(8'h00); // 0000_0000
    send_byte(8'h55); // 0101_0101
    send_byte(8'h01); // 0000_0001 — LSB boundary
    send_byte(8'h80); // 1000_0000 — MSB boundary
    repeat(4) @(posedge clk);
    $display("[DONE] All directed tests complete"); $finish;
  end

  initial begin
    $dumpfile("spi.vcd");
    $dumpvars(0, spi_tb_directed);
  end
endmodule
```

## Bugs Found and Fixed During Development

These were real issues encountered while writing the RTL — documented here as evidence of debugging process.

| Bug | Symptom | Root Cause | Fix Applied |
|-----|---------|------------|-------------|
| **MOSI 1-cycle lag** | First bit on MOSI appeared one clock late; slave would sample a stale zero | MOSI was only updated on `sck_en` inside SHIFT, so the first SCK rising edge saw bit 6, not bit 7 | Pre-load MOSI with `shift_reg_out[DATA_WIDTH-1]` in ACTIVATE_CS before any SCK activity |
| **Missing `clk_cnt`** | Design would not compile | `clk_cnt` register declared nowhere despite being used in the clock divider | Added `logic [$clog2(CLK_DIV)-1:0] clk_cnt` |
| **Wrong `bit_cnt` width** | Synthesis warning; potential counter overflow on 16-bit mode | `bit_cnt` was sized `$clog2(CLK_DIV)` instead of `$clog2(DATA_WIDTH)` | Corrected to `$clog2(DATA_WIDTH)` |
| **Missing FSM state register** | FSM was combinational only; state would glitch | The `always_ff` block for `state <= next_state` was absent | Added separate sequential block for state register |
| **Typo in state names** | Compilation error | `IDEL` written in three places instead of `IDLE` | Corrected all instances |
| **`miso_data` latched too early** | Received byte was incomplete (last bit missing) | Latching inside SHIFT state captured data before the final SCK edge | Moved latch to DEACTIVATE_CS where `shift_reg_in` is fully populated |

## Key Learnings

**1. MOSI pre-load is non-negotiable for SPI Mode 0.**  
In Mode 0 (CPHA=0), the slave samples MOSI on the *first* rising SCK edge. If MOSI is only updated inside the SHIFT state, that first edge sees whatever was on MOSI before — producing a one-bit offset in every transfer. The fix is to pre-drive MOSI with bit 7 during ACTIVATE_CS, one cycle before SCK starts toggling.

**2. Always separate FSM next-state logic from the state register.**  
Combining `always_comb` (next-state) and `always_ff` (state register) into one block is a common beginner mistake that either fails to compile or produces latches. Keeping them separate makes lint, synthesis, and debugging dramatically cleaner.

**3. Directed testbench first, UVM second — not the other way around.**  
A directed, non-UVM testbench with 6 known-good vectors catches RTL bugs in minutes. If UVM were introduced first, the same bugs would surface buried inside UVM error messages — much harder to isolate. Proving RTL correctness before adding infrastructure is a workflow that scales.

**4. `$clog2()` widths must be matched to the right parameter.**  
Using `$clog2(CLK_DIV)` for a counter that tracks *bit position* (should be `$clog2(DATA_WIDTH)`) silently passes compilation for the default case (both = 2 when CLK_DIV=4 and DATA_WIDTH=8) but would overflow at DATA_WIDTH=16. Parameterised RTL requires each width expression to be independently verified against its intent.

**5. Latch timing in SPI matters more than it looks.**  
`miso_data` must be latched in DEACTIVATE_CS, not at the end of SHIFT. The final MISO bit is captured into `shift_reg_in` on the last `sck_en` inside SHIFT — DEACTIVATE_CS is the first state where the register is guaranteed to hold the complete received byte. Latching one state early produces a byte that is always one bit short.
